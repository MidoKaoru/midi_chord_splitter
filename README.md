# MIDI Chord Splitter

1つのトラックに和音（複数音の同時発音）として入力されたMIDIデータを、音ごとに独立したトラックへ分割するブラウザ完結型のWebアプリケーションです。

DAWで作成したMIDIデータをMuseScoreなどの楽譜作成ソフトへインポートする際に、パート別のトラック構成にする目的で使用します。

## 主な機能

* **和音の自動分割**: 同タイミングで鳴っている音符を検出し、指定した数のトラックへ1音ずつ振り分けます。同タイミングの音は音高の高い順（降順）にトラック1・2・3…へ割り当てられます。
* **タイミング許容誤差**: 発音タイミングが厳密に一致していない場合でも、指定した tick 数以内のズレであれば同一和音として扱います（0 / ±10 tick / カスタム値から選択）。
* **発音中トラックの回避**: 前のタイミングから音を伸ばしているトラックには新たな音を割り当てず、空いているトラックへ順番に配置します。
* **同ピッチ重複音の正確な対応付け**: 同じ音高の音符が重複して鳴っている場合、LIFOスタックを用いて「後から鳴り始めた音が先に終わる」というMIDIの標準的な挙動に従い noteOn↔noteOff を正確にペアリングします。
* **オーバーフロー処理**: 和音の音数が分割トラック数を超過した場合、超過分は最終トラックに集約されます。実行後のプレビューで各トラックのノート数と超過件数を確認できます。
* **メタイベントの保持**: テンポ・拍子記号・調号などのメタイベントはテンポトラック（トラック0）に引き継がれ、分割後のファイルでも正しく再生されます。
* **ブラウザ完結**: すべての処理はローカルのブラウザ内で完結し、外部サーバーへのデータ送信は行いません。

## 注意事項

* **保持されるイベントについて**: 分割後の各トラックに含まれるのはノートイベント（noteOn / noteOff）のみです。プログラムチェンジ・コントロールチェンジ・ピッチベンドなどのイベントは分割トラックには引き継がれません。必要に応じてDAWまたは楽譜ソフト上で再設定してください。

* **出力フォーマットについて**: 入力ファイルのフォーマット（Format 0 / Format 1）によらず、出力ファイルは常にFormat 1で書き出されます。

* **テンポトラックの扱いについて**: Format 1 ファイルの場合、トラック0はテンポトラックとしてそのまま保持されます。Format 0 ファイルまたはトラック0を分割対象に指定した場合は、対象トラックからメタイベントを抽出してテンポトラックを生成します。

* **分割トラック数について**: 2〜9の範囲で指定してください。10以上の値は入力エラーになります。

## 内部の処理フロー

```
parseMidi() でバイナリ→オブジェクト変換
      ↓
対象トラックのイベントを絶対 tick 付きで展開
      ↓
LIFO スタックで noteOn↔noteOff を正確にペアリング
      ↓
許容誤差内の同タイミングペアをグループ化
      ↓
グループ内を音高（note番号）降順にソート
      ↓
発音中のトラックをスキップしながらトラックへ割り当て
（空きなしの場合は最終トラックへ集約）
      ↓
テンポトラック + 非対象トラック + 分割トラックを結合
      ↓
writeMidi() でバイナリ変換 → ダウンロード
```

### noteOn↔noteOff ペアリングについて

同一ピッチの音が重複して鳴っている場合、単純な「先頭探索」では以下のように noteOff が入れ替わります。

```
tick   0: noteOn  C4（長い音）
tick 240: noteOn  C4（短い音・重複）
tick 360: noteOff C4 ← 短い音の終わり
tick 480: noteOff C4 ← 長い音の終わり

単純先頭探索: noteOn@0 ↔ noteOff@360（誤）/ noteOn@240 ↔ noteOff@480（誤）
LIFOスタック: noteOn@0 ↔ noteOff@480（正）/ noteOn@240 ↔ noteOff@360（正）
```

本ツールではLIFOスタックを採用し、上記の問題を回避しています。
